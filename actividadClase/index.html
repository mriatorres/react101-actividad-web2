Introducción

Cuando empezamos a desarrollar un proyecto con un nuevo lenguaje o framework una de las primeras dudas que nos surgen es cómo organizo mi proyecto y, en el caso de React, esto no es una excepción.

Si nos basamos en la documentación oficial, React no define una forma correcta de organizar nuestros ficheros y lo deja a criterio del desarrollador. Esto puede parecer extraño si hemos trabajado anteriormente con frameworks como Angular, pero hay que tener en cuenta que React es una librería de frontend que se puede utilizar tanto para crear aplicaciones completas, como para componentes concretos dentro de una base de código ya existente, por lo tanto, tiene sentido que esta elección sea libre.

Anatomia de un proyecto

Estructura por tipo de fichero

Esta organización nos permite tener estructuras muy sencillas para proyectos que estén empezando o queramos realizar sin demasiados quebraderos de cabeza, y es la que recomendaría para todos aquellos que se inicien con React. El problema principal es que una vez empieza a crecer nuestro proyecto puede convertirse en un auténtico caos, sobre todo a nivel de componentes, ya que en la variante más sencilla los tendríamos todos dentro de una carpeta llamada componentes.

Estructura por funcionalidad o modular

La siguiente forma de organización que se menciona en la documentación es la basada en módulos, esta organización es interesante para proyectos con una escala mayor.

La idea principal es que cada módulo que definimos tenga todo el código relacionado con este y solo se importe código del modulo en sí. Cuando tenemos varios módulos que necesitan de la misma pieza de código esta la podemos escribir dentro de una carpeta compartida e importarla a los diferentes módulos. La regla fundamental que se pretenden seguir es no importar código entre módulos.

Ejemplo

src/ |-- components/ | |-- Avatar/ | | |-- Avatar.jsx | | |-- Avatar.test.js | |-- Button/ | | |-- Button.jsx | | |-- Button.test.js | |-- TextField/ | | |-- TextField.jsx | | |-- TextField.test.js |-- contexts/ | |-- UserContext/ | | |-- UserContext.js |-- hooks/ | |-- useMediaQuery/ | | |-- useMediaQuery.js |-- features/ | |-- Home/ | | |-- components/ | | | |-- SomeUserProfileComponent/ | | | | |-- SomeUserProfileComponent.jsx | | | | |-- SomeUserProfileComponent.test.js | | |-- utils/ | | |-- services/ | | |-- hooks/ | | |-- contexts/ | | |-- views/ | | | |-- HomeView.jsx | | |-- pages/ | | | |-- HomePage.jsx | |-- index.js |-- utils/ | |-- some-common-util/ | | |-- index.js/ | | |-- index.test.js |-- services/ | |-- some-common-service/ | | |-- index.js/ | | |-- some-common-service.js/ | | |-- index.test.js |-- App.jsx |-- index.js

Propuesta estructura

En este ejemplo:
La carpeta node_modules contiene las dependencias del proyecto.
La carpeta public contiene archivos estáticos como index.html y favicon.ico.
La carpeta src es donde se encuentra la mayor parte del código de la aplicación React. Contiene el punto de entrada index.js, el componente principal App.js, componentes adicionales en la carpeta components, estilos en la carpeta styles, y recursos como imágenes en la carpeta img.
El archivo package.json describe las dependencias y scripts del proyecto.
Los archivos webpack.config.js y babel.config.js podrían contener configuraciones personalizadas para Webpack y Babel si se han configurado de manera personalizada.

Componentes

¿Qué es un componente?

Un componente web es una parte de la interfaz de una página o aplicación web que podemos reutilizar. Por ejemplo, una línea de producto de un carrito de la compra, o un elemento colapsable.
Los frameworks o librerías como React se basan en el concepto de componente. De esta forma, todo lo que vamos a crear son componentes que iremos usando para crear la interfaz deseada. Los componentes de React, además, pueden personalizarse a través de un mecanismo llamado props, que no es otra cosa que pasarle valores a través de los atributos del componente HTML.

Importancia

Los componentes son una parte fundamental en React y desempeñan un papel central en el desarrollo de aplicaciones en esta biblioteca:

  • Reutilización de código: Los componentes permiten encapsular el comportamiento y la interfaz de usuario en unidades lógicas y reutilizables. Puedes usar un componente en múltiples lugares de tu aplicación, lo que ahorra tiempo y esfuerzo en la escritura de código repetitivo.

  • Abstracción de la interfaz de usuario: Los componentes dividen la interfaz de usuario en piezas más pequeñas y manejables. Esto facilita el mantenimiento y la comprensión del código, ya que cada componente se centra en una sola responsabilidad.

  • Separación de preocupaciones: React promueve la separación de preocupaciones al dividir la lógica de presentación (UI) de la lógica de negocio. Esto hace que tu código sea más organizado y mantenible al permitir que diferentes equipos trabajen en diferentes aspectos de la aplicación.

  • Facilita la colaboración: Los componentes bien definidos facilitan la colaboración entre desarrolladores, ya que cada componente puede ser desarrollado y probado de manera independiente. Esto es especialmente útil en equipos grandes.

  • Rendimiento optimizado: React utiliza un mecanismo de reconciliación virtual para minimizar las actualizaciones del DOM, lo que mejora el rendimiento de la aplicación. Los componentes funcionales pueden optimizarse aún más mediante el uso de React.memo y React.useMemo.

  • Pruebas unitarias: Los componentes aislados son más fáciles de probar de manera unitaria. Puedes escribir pruebas para cada componente para garantizar su correcto funcionamiento sin depender de otras partes de la aplicación.

  • Facilita la escalabilidad: A medida que tu aplicación crece, los componentes te ayudan a mantener un código más organizado y escalable. Puedes agregar nuevos componentes o reemplazar componentes existentes sin afectar otras partes de la aplicación.

  • Ecosistema de componentes reutilizables: React cuenta con una comunidad activa y una gran cantidad de bibliotecas y frameworks que ofrecen componentes preconstruidos y reutilizables que puedes integrar en tu aplicación.

En resumen, los componentes en React son fundamentales para la construcción de aplicaciones web modernas, ya que promueven la reutilización, la organización, la escalabilidad y el rendimiento del código. Al adoptar la filosofía de componentes, puedes crear aplicaciones más mantenibles y eficientes.

Tipos de componentes

En React, puedes crear componentes tanto como funciones (componentes funcionales) o como clases (componentes de clase). Cada enfoque tiene sus propias características y casos de uso. Aquí te proporciono una descripción de ambos tipos de componentes:

Componentes Funcionales
  • Componentes Funcionales Básicos: Los componentes funcionales son simplemente funciones de JavaScript que devuelven elementos de React (usualmente JSX). Son simples y fáciles de entender.

    Hola, soy un componente funcional.
    ; } export default MiComponenteFuncional;
  • Componentes Funcionales con Estado (Hooks): Los componentes de clase pueden tener estado local y métodos de ciclo de vida, como `componentDidMount`, `componentDidUpdate`, etc.

    button onClick={() => setCount(count + 1)}>Incrementar /div> ); } export default Contador; ```

Componentes de Clase
  • Componentes de Clase Básicos: Los componentes de clase son clases de JavaScript que extienden la clase `React.Component`. Antes de la introducción de los Hooks, eran la forma principal de manejar el estado y los ciclos de vida.

    ```javascript import React, { Component } from 'react'; class MiComponenteDeClase extends Component { render() { return div>Hola, soy un componente de clase./div>; } } export default MiComponenteDeClase; ```

  • Componentes de Clase con Estado: Los componentes de clase pueden tener estado local y métodos de ciclo de vida, como `componentDidMount`, `componentDidUpdate`, etc.

    ```javascript import React, { Component } from 'react'; class Reloj extends Component { constructor(props) { super(props); this.state = { time: new Date() }; } componentDidMount() { this.timerID = setInterval(() => this.actualizarHora(), 1000); } componentWillUnmount() { clearInterval(this.timerID); } actualizarHora() { this.setState({ time: new Date() }); } render() { return (

    Hora actual: {this.state.time.toLocaleTimeString()}

    ); } } export default Reloj; ```

A partir de la versión 16.8 de React, se introdujeron los Hooks, lo que permitió a los componentes funcionales manejar el estado y los efectos de manera más sencilla. Se recomienda el uso de componentes funcionales con Hooks en lugar de componentes de clase, a menos que necesites mantener o trabajar en una base de código existente que utiliza componentes de clase. Los componentes funcionales son más concisos y fáciles de entender en la mayoría de los casos.


Tanto las funciones como las clases en React tienen sus propias características y casos de uso. A continuación, se presentan las características clave de ambos enfoques:

Características de las Funciones en React (Componentes Funcionales)

  • Simplicidad: Los componentes funcionales son más concisos y fáciles de entender en comparación con las clases. Son una opción excelente para componentes simples y reutilizables.

  • Hooks: Los componentes funcionales pueden utilizar Hooks para manejar el estado y los efectos, lo que facilita la gestión de la lógica compleja dentro de un componente funcional.

  • Sintaxismás ligera: La sintaxis de un componente funcional es más ligera y moderna, lo que lo hace más atractivo para muchos desarrolladores.

  • Rendimiento: Los componentes funcionales pueden ser más eficientes en términos de rendimiento en comparación con las clases, ya que no tienen el overhead de las instancias de clase.

  • Mejor soporte para React Concurrent Mode: Los componentes funcionales están más alineados con los conceptos de Concurrent Mode, una característica futura de React para mejorar el rendimiento y la experiencia del usuario

  • Más adecuado para bases de código existentes: Si estás trabajando en una base de código existente que utiliza clases, puede tener sentido seguir utilizando clases para mantener la consistencia en el código.
